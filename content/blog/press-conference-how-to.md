大きな発表をするとき、プレスリリースだけでなく記者会見・発表会という選択肢があります。記者に直接伝え、質疑応答で深く理解してもらい、その場の熱量を記事に乗せてもらう――うまく運営できれば、露出のインパクトは大きくなります。

一方で、準備不足や集客失敗のリスクもあります。この記事では、<span class="highlight-yellow">記者会見・発表会の開き方を、開催判断から準備、当日運営、フォローまで</span>実務目線で詳しく解説します。記者を招いて価値ある場を作るために必要なすべての情報をまとめました。

## 記者会見を開くべき場面・開かなくてよい場面

記者会見は準備に相当のリソースがかかります。最初に「開くべき発表かどうか」を冷静に判断することが大切です。

### 開く価値がある場面

- **事業に大きなインパクトのある発表**：新規事業立上げ、大型資金調達、上場、大型M&A、主要パートナーシップ締結
- **社会的に関心の高いテーマ**：社会課題への取り組み、業界を変えるような新技術・サービス
- **質疑応答が必要な内容**：複雑な内容・誤解を防ぎたい内容・記者が深く理解する必要があるもの
- **ビジュアルや体験で伝えたい**：製品発表会、実演が伴うもの、フォトセッションの価値があるもの
- **権威性・信頼性を高めたい**：創業者・CEOが直接語る必要のある重要な発表

### プレスリリースで十分な場面

- **日常的な小さな発表**：新機能追加、定期的な受賞報告、軽微な人事異動
- **ニュース性が限定的な内容**：社内向けのお知らせに近いもの
- **記者に来てもらうほどの新規性がない**：「業界で珍しくない」ニュース

記者会見は記者の時間を拘束します。「わざわざ来てもらう価値があるか」を冷静に判断しましょう。配信で済む場合の考え方は [プレスリリース配信サービスの選び方](/blog/press-release-distribution-comparison) を参照してください。

<div class="callout-warn"><p>⚠️ 注意: 記者会見に呼んでも、来てもらう価値のない発表だと判断されると次から案内を無視されます。記者との長期的な関係を損なわないよう、開催基準を社内で明確にしておきましょう。</p></div>

## 準備のステップ

記者会見の準備は、開催日の2〜4週間前から始めるのが目安です。規模によってはさらに前から動く必要があります。

### ステップ1：企画・メッセージ設計

**キーメッセージを1つに絞る**

記者会見で最も伝えたいことは何か。「一言で言うと？」という問いに答えられるメッセージを設計します。伝えたいことが複数ある場合も、最優先のメッセージを1つ決め、それを全体の軸にします。

**発表内容の整理**

- 発表の背景・文脈（なぜ今これを発表するのか）
- 発表の内容（具体的な事実・数字）
- 期待されるインパクト（社会・業界・顧客への意味）
- ビジュアル・デモの要否

**登壇者の決定**

- 誰が登壇するか（CEO・事業責任者・提携先の代表など）
- 役割分担（主発表者・質疑対応者）
- 登壇者が記者に与える印象・信頼性

**想定問答（QA）の洗い出し**

記者が聞きそうな質問をあらかじめリストアップし、回答方針を決めておきます。特に「答えにくい質問」への準備が重要です。

### ステップ2：日時・会場の選定

**日時の選び方**

| 観点 | ポイント |
|---|---|
| 曜日 | 火〜木が最も記者が来やすい。月曜・金曜は避けるのが基本 |
| 時間帯 | 午前10時〜11時、午後2時〜3時が多い。記事締切前の遅い時間は避ける |
| 他の大型発表 | 同業他社・政府・有名企業の発表と重なっていないか確認 |
| 記念日・祝前日 | 翌日が祝日だと記者のモチベーションが下がる傾向 |

プレスリリース配信のタイミングの考え方（→ [プレスリリース配信タイミング](/blog/press-release-timing)）も参考になります。

**会場の選定基準**

- **アクセス**：主要なメディアが集中するエリア（東京なら大手町・霞が関周辺）に近いか
- **収容人数**：想定参加者数の1.5倍程度の余裕を持たせる
- **配信設備**：マイク・プロジェクター・ライブ配信設備の有無
- **受付スペース**：記者の受付・名刺交換できるスペースがあるか
- **フォトセッションスペース**：写真撮影に適した背景・照明
- **セキュリティ**：エンバーゴ（解禁前情報）がある場合の機密管理

<div class="callout-tip"><p>✅ 実践ポイント: 会場の下見は必須です。音響・照明・ネット接続・電源の位置を事前に確認し、当日のセッティング時間を十分確保してください。特にハイブリッド開催の場合は配信環境の事前テストが不可欠です。</p></div>

### ステップ3：案内・メディア集客

**メディアリストの準備**

招待するメディアのリストを作成・更新します（→ [メディアリストの作り方](/blog/media-list-building)）。発表内容に関連する記者・編集者を優先的に招待します。

**招待状の送付**

- 開催2〜3週間前に最初の案内を送る
- 招待状に含める情報：日時・場所・内容の概要・登壇者・RSVP方法・問い合わせ先
- 出欠確認（RSVP）の締切を設ける

**リマインダーと集客フォロー**

- 開催1週間前：第1リマインダー
- 開催前日：第2リマインダー（会場への道案内・当日の流れを含む）
- 返答がない重要メディアには個別にフォローコール

「報道関係者各位」という一斉案内だけでなく、重要メディアには個別に「ぜひ来てください」という一声を添えると集客率が上がります。

### ステップ4：資料・配布物の準備

**プレスキットの作成**（→ [プレスキットの作り方](/blog/press-kit-how-to-make)）

- プレスリリース本文
- 会社概要・ファクトシート
- 製品・サービスの説明資料
- 写真・ロゴデータ（高解像度）
- 担当者の連絡先

**会場配布物の準備**

- プレスキット（印刷物またはUSBメモリ、デジタル配布のQRコード）
- 名刺交換のための記帳台・名刺入れ
- 飲み物・軽食（特に長丁場の発表会の場合）

**プレゼンテーション資料（スライド）**

- フォントサイズは大きめ（最後列からも読める）
- スライドより「話している人」に注目してもらう意識を
- 写真・グラフィック・動画を活用して視覚的に訴える

## 当日の運営

### タイムライン例（2時間の発表会の場合）

| 時刻 | 内容 |
|---|---|
| 開始1.5時間前 | 会場設営・音響テスト・配信テスト |
| 開始30分前 | 受付開始・メディア誘導 |
| 開始5分前 | 登壇者待機・最終確認 |
| 0:00 | 開会・司会者の開会宣言 |
| 0:05〜0:40 | 発表（登壇者によるプレゼンテーション） |
| 0:40〜1:10 | 質疑応答（QA） |
| 1:10〜1:30 | フォトセッション（写真撮影タイム） |
| 1:30〜 | 個別取材・懇談 |

### 進行台本の作成

開会から閉会まで、司会が読み上げる台本を作成します。

- 開会：参加へのお礼・当日の流れの説明
- 登壇者紹介：肩書き・氏名を正確に
- 質疑応答の仕切り：「ご質問のある方は手をお挙げください」
- 時間管理：質問1問あたりの目安時間

### 登壇者の準備

登壇者は事前に十分な練習が必要です（→ [取材対応の基本](/blog/media-interview-preparation)）。

- 話す内容のポイントを整理（台本の丸読みは避ける）
- 想定問答をもとにQA練習（難しい質問・攻撃的な質問への対応）
- 立ち位置・マイクの使い方・カメラへの向き方

### 受付・誘導の体制

- 受付スタッフ：名刺取得・参加者リストへの確認
- 誘導スタッフ：トイレ・写真撮影エリアへの案内
- クローク係（長い取材かばんなどの預かり）

### 質疑応答（QA）の運営

QAは記者会見の中でも最も重要で、最もリスクがある場面です。

**QA運営のポイント**

- 司会者が手をあげた記者を指名する（混乱を避ける）
- 1問ずつ丁寧に回答する（まとめ質問は避ける）
- 回答できない質問（情報開示前、法的に答えられないなど）は「現時点ではお伝えできません」と明確に
- 長引く場合は「次の方のご質問をどうぞ」と切り上げる

**難しい質問への対応例**

| 質問タイプ | 対応の基本方針 |
|---|---|
| 数字・根拠を求める質問 | 持っているデータで答える。ない場合は「後ほど確認します」 |
| ネガティブな前提の質問 | 前提を丁寧に訂正した上で回答 |
| 競合との比較質問 | 競合を批判しない。自社の強みで答える |
| 答えられない質問 | 「現時点では開示できません」と明確に |

## オンライン・ハイブリッド開催のポイント

コロナ禍以降、オンライン・ハイブリッド開催が一般化しました。

### オンライン配信の準備

- **配信プラットフォーム**：Zoom Webinar・YouTube Live・Vimeoなど
- **回線**：有線LAN必須。Wi-Fiのみでの本番配信は避ける
- **音声**：マイクの品質が映像以上に重要。収音エリアと音漏れを確認
- **カメラ**：複数カメラで切り替えられると視聴者の飽き防止になる
- **リハーサル**：本番と同じ環境で事前テストが必須

### ハイブリッド開催の注意点

- オンライン参加者が置いてきぼりにならないよう、会場の様子が伝わるカメラ配置を
- 質問受付：オンライン参加者はチャット・Q&Aツールで受付し、司会が読み上げる
- 録画配信：後から見られる録画の公開可否を事前に決めておく

### オンライン参加者への配慮

- 開始前のアナウンス：接続確認・音声確認
- 質問の機会を均等に：オンラインと会場の質問をバランスよく取り上げる
- 資料の事後送付：オンライン参加者には画面越しでは見にくい資料を後送

## 開催後のフォロー

記者会見は開催して終わりではありません。その後のフォローで効果を最大化します。

### 参加できなかった記者へのフォロー

- 当日来られなかった記者へプレスキットを送付
- 個別取材の申し出があれば積極的に対応

### 掲載状況の確認と集計

<span class="highlight-yellow">どのメディアに・どのような内容で・どのトーンで報じられたか</span>を確認します（→ [PR効果測定の基本](/blog/pr-effectiveness-measurement)）。

- リリース当日から数日間の掲載状況をモニタリング
- 掲載媒体・記事URL・リーチ（閲覧数の目安）を記録
- ネガティブな報道があった場合は早期対応を検討

競合の同種の発表と比べてどうかを把握すると、次回の改善につながります。競合のプレスリリースを自動で追うなら [ReAnker](/) が便利です（月額300円、無料プランあり）。

### 来場した記者との関係継続

- お礼メールを翌日中に送る
- 記事が掲載されたらコメントや感謝の連絡
- 次の発表の際のメディアリストに確実に追加

<div class="callout-info"><p>💡 ポイント: 記者との関係は一度の記者会見で作るものではなく、積み重ねて育てるものです。発表がなくても業界情報の提供・コメント協力など、日常的なコミュニケーションを通じて信頼関係を築きましょう。</p></div>

## 記者会見の成功指標

| 指標 | 測定方法 |
|---|---|
| 参加者数 | 出席した記者・メディアの数 |
| 掲載媒体数 | 発表から1週間で掲載された媒体数 |
| リーチ | 掲載媒体の読者数・PVの合計 |
| トーン分析 | ポジティブ・ニュートラル・ネガティブの比率 |
| 問い合わせ件数 | 発表後の問い合わせ・商談の増加数 |

## まとめ

記者会見・発表会は、開催する価値を見極めたうえで、メッセージ設計・会場・集客・資料を準備し、当日の進行とQAを固め、開催後にフォローする――この流れで、大型発表のインパクトを最大化できます。記者の時間に見合う価値を提供することを常に意識し、長期的なメディア関係構築の一環として位置づけましょう。
