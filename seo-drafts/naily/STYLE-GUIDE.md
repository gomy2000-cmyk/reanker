# ナイリー（nAIly）SEO記事 制作スタイルガイド（執筆共通ルール）

このファイルは、初期5本（001〜005）と同じ品質・体裁で残り記事を量産するための共通ルールです。
**新しい記事を書くときは、必ずこのガイドと `drafts/001-company-dedicated-ai-chat.md` を参照してから書いてください。**

---

## サービスの立ち位置（ナイリー / nAIly）

会社名・サービス名・ブランド名・相談窓口名など、**好きな名前でAIチャット環境を持てるサービス**。

- 好きな名前でAIチャットを使える／専用URLを持てる／ロゴ設定ができる
- 「自社専用AIチャット」のように見せられる／「○○相談窓口」を作れる
- 最短5営業日程度で利用開始できるイメージ／低コスト／中小企業でも導入しやすい
- 「AI導入が難しい・面倒・何から始めればいいか分からない」会社向け

**前面に出しすぎない**：RAG／高度な社内文書検索／複雑なAI開発／大企業向け重厚システム／完全自動対応／全問い合わせをAIで解決のような過剰表現。

ポジション＝「AI導入」「AIチャットボット開発」ではなく、**“好きな名前で使える、自社専用風のAIチャット環境を手軽に持てるサービス”**。

## トーン

初心者向け／やさしい／実務的／中小企業の担当者に寄り添う／断定しすぎない／でも曖昧すぎない／営業感を出しすぎない。専門用語は必ずやさしく補足。**人間が書いたように自然に**（出だし・文長を変え、AIっぽい過剰に整った言い回しを避ける）。

- **避ける語**：革新的な／最先端の／未来を変える／圧倒的に／完全自動化／誰でも爆速で／すべて解決／DXを加速／次世代AIソリューション
- **使ってよい語**：まずは小さく始める／会社名で使える／自社専用のように使える／専用URLで案内できる／ロゴを設定できる／相談窓口として使える／AI導入の最初の一歩／難しい開発をせずに始める／中小企業でも導入しやすい
- 過剰な自動化・万能感を出さない。競合サービス名の断定的批判はしない。各AI（ChatGPT/Claude/Gemini）の機能・価格を具体的に断定しない（一般的・正確な範囲で）。

## 文字数

本文 **4,000〜6,000字**（日本語・レンダリング後）。基本 4,500〜5,000字を狙う。短すぎは不可。

## 体裁（001を必ず踏襲）

1. **YAMLフロントマター**（キー名を厳守）：
   ```
   ---
   primary_kw: <KW>
   secondary_kw: [..., ...]
   search_intent: <文>
   target_reader: <文>
   funnel_stage: <認知|比較検討|導入検討|CV直前>
   article_type: <基礎解説|導入ガイド|比較|費用|業種別|悩み解決|使い方|相談窓口化|各AI別解説 など>
   estimated_word_count: <数値>
   cta_policy: <文>
   internal_links:
     - /blog/<slug>
   image_count: <実際のプレースホルダ数>
   status: draft
   ---
   ```
2. **H1タイトル → リード文 → CTA（上部）**。
3. H2/H3は内容に応じて自然に（H2数は固定しない）。**表を2〜3個**（比較・選択肢・メリット整理に積極活用）。**FAQ的セクション**を入れる。
4. **CTAは4箇所**、ラベルを厳守：`（上部）`『関連セクション直後＝（セクション）』`（中盤）``（下部）`。形式：
   ```
   > **💬 CTA（上部）**
   > <1〜2行。ナイリーを“名前・ブランド・専用URL・ロゴで使える軽量AIチャット環境”として自然に紹介>
   > 👉 [<ラベル>](https://naily.example.jp/)（仮リンク）
   ```
   CTAは高機能AIシステムではなく**手軽さ**を訴求。過剰な約束をしない。文面は記事ごとに変える。
   - **CTAで避ける**：今すぐ絶対導入すべき／完全自動化できます／すべての問い合わせを解決／社内文書をすべて学習／高度なRAG／大企業向けAI基盤。
5. **画像プレースホルダ**＝HTMLコメント、本文約1,500字に1枚（3〜4枚）：
   ```
   <!-- 🖼 IMAGE img-AXX-0N ｜ <用途> ｜ 詳細は image-prompts/0NN-<slug>.md -->
   ```
   フロントマターの `image_count` を実数に一致させる。
6. **内部リンク**＝指定の `/blog/<slug>` を本文に自然に挿入。
7. 末尾に**まとめ**（箇条書き）→ **CTA（下部）**。

## メタJSON（metadata/0NN-<slug>.json）

キー：`title, meta_title, meta_description, primary_kw, secondary_kw[], slug, category, tags[], funnel_stage, target_reader, cta_type, internal_link_suggestions[], image_prompt_file, draft_file, status`。

- `meta_title`：32全角字前後・KWを前方に・不自然な詰め込みはしない
- `meta_description`：90〜120字・検索意図に答えつつナイリー訴求を軽く
- 有効なJSON（UTF-8）。`status` は `"draft"`。

## 画像案（image-prompts/0NN-<slug>.md）

- 冒頭に「対象記事・画像枚数」、続けて**「共通デザイン仕様は image-prompts/001-company-dedicated-ai-chat.md を参照」**と明記し、禁止リストとブランドカラー要確認を再掲。
- 画像ごとに：**用途／画像サイズ／比率／背景／余白／色数／デザインテイスト／入れる要素／入れない要素／禁止表現／生成プロンプト（英語・テキストなし土台用）**。
- 枚数は本文プレースホルダ数と一致。
- **禁止（必須）**：グラデーション／3D／ネオン／サイバーパンク／近未来／過剰な光・粒子・ホログラム／青紫の派手な発光／人物の強い登場／いかにもAI生成っぽい質感。
- 許容：薄い黄色アンダーライン／控えめなボックス／比較表／チェックリスト／シンプルカード／UIモック風／フラット図解。
- 仮配色：オフホワイト背景＋ダークグレー文字＋落ち着いたブルー＋控えめな黄色（**正式ブランドカラー確定後に差し替え**）。日本語ラベルはデザインツールで後載せ推奨。
