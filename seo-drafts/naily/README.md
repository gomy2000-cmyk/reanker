## 1. 今回の作業概要

ナイリー（nAIly）のSEO記事を大量制作するための「下準備」と「初期記事下書き」を作成しました。

ナイリーは、会社名・サービス名・ブランド名・相談窓口名など、**好きな名前でAIチャット環境を利用できるサービス**です。企業が自社専用のAIチャット環境を、短期間・低コストで持てることを訴求します。本パックは、その方針に沿ったSEOコンテンツの土台です。

本パックに含まれるもの：

- 初期SEO対策候補 **50キーワードの整理・クラスタリング**
- カニバリ（重複）を避けた **15記事の制作計画**
- 優先度上位 **5記事の本文下書き**（各 約4,000〜5,000字）
- 5記事分の **メタ情報（JSON）**
- 5記事分の **画像案・画像生成プロンプト**

## 2. 保存先と、その判断理由

**保存先：`/seo-drafts/naily/`（リポジトリ直下）**

### なぜここに保存したか

リポジトリ構造を確認した結果、以下の理由から、本番に影響しない独立ディレクトリを選びました。

1. **`content/blog/` は自動公開される**
   `lib/blog.ts` が `content/blog/` 内の全 `.md` を読み込み、`app/blog/[slug]/page.tsx`・`app/sitemap.ts` で一覧・サイトマップに自動掲載します。ここに下書きを置くと、**公開日（publishedAt）次第で本番サイトに勝手に出てしまう**ため、避けました。
2. **このリポジトリは別サービス「ReAnker（reanker.com）」のもの**
   ReAnkerは競合監視SaaSで、ナイリーとは別サービスです（ブログCTAやメタも全てReAnker固定）。ナイリーのSEOパックは、混線を避けて**独立した場所**に置くのが安全です。
3. **依頼の指示（既存の公開ページ・ルーティング・デザイン・本番コードを変更しない）に最も忠実**
   `/seo-drafts/naily/` なら、既存のビルド・公開・ルーティングに一切影響しません。

> 参考：依頼書の保存先候補（`/seo-drafts/naily/`、`/content-drafts/naily/`、`/docs/seo/naily/`、`/articles/drafts/naily/`）のうち、最も意図が明確な `/seo-drafts/naily/` を採用しました。

### ⚠️ 重要：このパックを本番ブログに載せる際の注意

- 将来このリポジトリ（ReAnker）の `content/blog/` に流用するのは**非推奨**です（別サービスのため）。ナイリーの実サイト／CMSへ移してください。
- 本パックのMarkdownは、移行先に合わせて**フロントマターの項目名を変換**してください。
  - 参考：ReAnkerブログのフロントマターは `title / description / ogTitle / publishedAt / updatedAt / category / tags`。本パックは制作管理用に `primary_kw / secondary_kw / search_intent / ...` を採用しています（移行時に対応付けが必要）。

## 3. 既存コードに触っていないことの明記

- **既存の公開ページ・ルーティング・デザイン・本番コードは一切変更していません。**
- 変更・追加したのは `/seo-drafts/naily/` 配下の**新規ファイルのみ**です。
- `app/`・`components/`・`lib/`・`content/blog/`・`public/` などの既存ファイルには触れていません。

---

## 4. Notion未接続のため、Git内に保存しました

- この環境は **Gitには接続されていますが、Notionには接続されていません**。
- そのため、依頼方針どおり **Notion APIやNotionデータベースへの出力は行わず**、リポジトリ内に **Markdown / CSV / JSON** で保存しました。
- 将来Notionへ取り込む場合は、`keyword-map.csv`（KW台帳）・`metadata/*.json`（記事メタ）・`drafts/*.md`（本文）がそのまま素材になります。

## 8. 次に人間が確認すべきこと

1. **保存先・方針の妥当性**：`/seo-drafts/naily/` で問題ないか。本番サイト／CMSへ移す前提でよいか。
2. **ナイリーの正式情報の確定**（仮置きしている箇所の差し替えに必要）
   - **LPの正式URL**：本文・CTAは `https://naily.example.jp/`（仮）で記載。要差し替え。
   - **各記事の公開URL（slug）**：`/blog/<slug>` で仮設定。実サイトの構造に合わせて調整。
   - **ブランドカラー・トンマナ**：画像案は「落ち着いた仮配色」。正式カラー確定後に差し替え。
3. **表現・事実の確認**：
   - 「最短5営業日程度」「数営業日」などの**リードタイム表現**が、実際のサービス提供条件と合っているか。
   - 「費用を抑えやすい」などの**費用感の表現**が、実際の料金体系と矛盾しないか。
   - 入力データの扱い（FAQで「導入前に確認を」と促進）の記載が、ナイリーの実仕様と整合するか。
4. **過剰表現の最終チェック**：完全自動化・全文書学習・高度RAG・大企業向け基盤などを**あおっていないか**（方針どおり避けて執筆済みですが、最終確認を推奨）。
5. **画像の作成方針**：画像は未生成（プロンプトのみ）。AI生成は日本語テキストが崩れやすいため、**図中ラベルはデザインツールで後載せ**を推奨（image-prompts に明記）。

## 10. 補足メモ

- 文字数：初期5本はいずれも本文 **約4,000〜5,000字**（依頼の「通常記事4,000〜6,000字」に準拠）。
- CTA：各記事に**上部・中盤・下部＋関連セクション**のCTAを配置。文面は `> 💬 CTA` のブロックで明示（移行先のCTAコンポーネントに差し替え可能）。
- 画像プレースホルダ：本文中に `<!-- 🖼 IMAGE img-Axx-xx ... -->` 形式でコメント挿入（レンダリングには出ません）。対応する仕様は `image-prompts/` にあります。
- カニバリ対策：空白違い・語順違いの同義KW（例：`AI相談窓口`＝`AI 相談窓口`）は必ず同一記事に統合済み。詳細は `keyword-clusters.json` を参照。
